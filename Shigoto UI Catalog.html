入力部品

業務UIの基礎部品。default / hover / focus / disabled / loading / error の状態を全部品で揃えるのが原則。

フォーム周辺

入力部品をまとめ、必須/任意・ヒント・エラーを管理する仕組み。氏名・フリガナ・会社情報は日本固有の事情を吸収する専用部品。

マスタ選択

顧客・商品・担当者・地域・年度——業務UIの背骨である「マスタからの選択」を、検索 + 直近履歴 + 詳細プレビューの組み合わせに統一する。

一覧・表示

業務UIの大半を占める「一覧 → 詳細」の動線。テーブル・カード・モーダル・検索・ページ送り・空状態 で完結する。

検索・絞込・並替

大量データを扱う業務系では、検索パネル / ファセット / ソート / 一括操作を 1セットで設計するのが定石。条件は URL に同期して共有可能に。

業務向け部品

金額・税率・郵便・住所・電話・日付——日本の業務入力で必ず手戻りする項目を、Web Component 1つずつに切り出す。

帳票・申請ヘッダー

請求書・見積書の詳細画面、および送信前の確認パネル。状態に応じた操作ボタンの出し分けと、差分のハイライトを担う。

承認・履歴

承認フロー・コメント・操作監査・差分。改ざん検知や監査対応も視野に、変更を全て可視化する。

ファイル・CSV

CSVは今もって業務系の最重要I/F。文字コード自動判定・ヘッダマッピング・エラー行の特定までを 1セットでカバーする。

帳票・印刷

A4縦・1ページに収める設計と、操作系を `@media print` で切り離す印刷プレビュー。納品物は今もって紙で残ることが多い。

ナビゲーション・構造

画面の中の移動・段階・分割を司る部品。タブで横切替、アコーディオンで縦折り、ステッパーで進行、パンくずで居場所、ドロワーで詳細をその場に重ねる。

選択・補完

マスタからの「選ぶ + 探す + 新規追加」をひとつの体験に。コンボボックスは静的リスト + 自由入力、オートコンプリートはサーバ側に都度問い合わせる動的検索。

状態・通知

「待っている」「補足する」「知らせる」のための部品群。ツールチップ・スピナー・スケルトン・バナー・トーストを 1 セットで揃える。

組織・権限・監査

組織図 → ロール → 監査の縦串。SaaS で必ず必要になる「誰が、何を、できる／したか」をワイヤーフレームレベルで定型化する。

業務向け部品（追加）

日本の業務UIに不可欠の固有概念。営業日換算・締日と支払サイト・全銀フォーマット・法人番号・印影。一般的な部品ライブラリでは取りこぼされる領域。
