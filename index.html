Why Shigoto UI

海外UIライブラリの "もう一歩手前" を埋める。

洗練された海外発のUIライブラリは、ボタン、テーブル、フォームを美しく揃えてくれます。 けれど、日本の業務システムには独自の前提があります。これらは「あとから足す」飾りではなく、設計の初日から組み込まれていなければならない要件です。

¥・カンマ・税率・全角数字・和暦—— 毎回プロジェクトごとに作り直されている "業務UIあるある" を、ライブラリ側で標準化する。

Shigoto UI は、見積・請求・承認・監査・帳票の全フェーズで使い回しできる71の Web Components を、トークンベースのフラットなワイヤーフレーム美学で揃えています。長時間の業務利用に耐える視認性、印刷を前提にした A4 レイアウト、変更を全て可視化する承認・監査の部品まで、業務系SaaSがほしい場所に手が届きます。

  • T+13桁 適格請求書（インボイス）登録番号形式チェック・接頭辞自動付与・国税庁API連携を内包。
  • 法人番号 13桁の登録番号から商号・本店所在地を国税庁の登録情報から自動取得。
  • 全銀 金融機関コード／支店コード／普通・当座／口座名義カナを1コンポーネントで一括管理。
  • 和暦／FY 4月始まりの年度・令和表記を切替可能。`startMonth`で年度開始月を変更。
  • ¥／半角 全角→半角の自動正規化、カンマ自動付与、税率別小計／合計を業務入力レベルで吸収。
  • 締日 月末・20日・10日締＋支払サイト・営業日換算まで標準化。
Why use it

使うべき4つの理由

日本の業務UIに必要なものを最初から備え、フレームワーク非依存で動き、印刷・監査までカバーする。

01 ／ Coverage

日本の業務に最適化された71コンポーネント

金額・税率・郵便・住所・電話・日付・法人番号・銀行口座・年度——日本固有の入力ルールを部品レベルで吸収。海外UIライブラリでは取りこぼされる "業務系の前提" を、最初から実装済みです。

money-input invoice-id corp-number bank-account fiscal-year
02 ／ Compatibility

Web Components first / フレームワーク非依存

Vanilla JS / React / Vue / Svelte、どんな環境でも <sgt-money-input> の1行で導入できる。標準仕様準拠なので、既存システムへの段階的な置換・併存・移行が容易です。

Custom Elements Shadow DOM React wrapper Vue wrapper
03 ／ Aesthetic

業務系のためのワイヤーフレーム美学

影・派手な色・無駄なアニメーションを排し、長時間の業務利用に耐えるフラットな視認性。CSS変数でテーマ・配色・余白・角丸を上書きでき、ブランドへの組み込みも 30 秒。

CSS variables flat / wireframe tabular-nums
04 ／ Lifecycle

見積→請求→承認→監査まで通貫

入力部品から帳票ヘッダ、明細テーブル、承認ステッパー、コメント・操作監査・差分表示、A4 1ページに収まる印刷プレビューまで。業務アプリの全フェーズを1ライブラリでカバーします。

Install

3行で導入。

npm でインストールするか、CDN を 1 行読み込むだけ。 Custom Elements として登録されるので、HTML に書くだけで使えます。 React / Vue 用のラッパパッケージも用意。

  1. @shigoto-ui/elements を読み込む
  2. HTML / JSX / template に <sgt-*> を書く
  3. CSS変数でブランドに合わせて色を上書き
HTML copy
<!-- 1行で導入 --> <script type="module" src="https://unpkg.com/@shigoto-ui/elements"></script> <!-- 業務UIの部品をHTMLにそのまま書ける --> <sgt-money-input value="1234567"></sgt-money-input> <sgt-postal-code-input></sgt-postal-code-input> <sgt-invoice-id-input required></sgt-invoice-id-input> <!-- イベントもネイティブ --> <script> document.querySelector('sgt-money-input') .addEventListener('sgt:change', e => { console.log(e.detail.value); }); </script>

業務システムを、より速く、より正確に。
